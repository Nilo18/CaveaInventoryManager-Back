# აღწერა
მოცემული პროექტი არის კავეა პლუსის ინვენტარის მენეჯმენტის სისტემის backend

## გამოყენებული ტექნოლოგიები
- Node.js v24.11.1
- Sequelize v6.37.7
- TypeScript
- Express.js v5.1.0
- PostgreSQL

## პროექტის სტრუქტურა
### src დირექტორია
root დირექტორიას წარმოადგენს src ფოლდერი, რომელშიც წარმოდგენილია პროექტის ფუნქციონალთან დაკავშირებული ყველა კომპონენტი. მოიცავს controllers, db, middleware, routes და scripts ქვედირექტორიებს.

### src/index.ts
აპლიკაციის საწყისს წარმოადგენს src/index.ts ფაილი, 
რომელშიც მითითებულია CORS სტანდარტის დეტალები, კერძოდ, სერვერი მოთხოვნას იღებს მხოლოდ localhost:4200 დან, საიდანაც იგი ელოდება front–ის მხრიდან კავშირს. გარდა ამისა, index.ts ფაილი პროექტს აკავშირებს PostgreSQL ბაზასთან, 
სადაც მას ეგულება არსებული inventory და location მოდელები. 

### src/controllers
აქ მოთავსებულია ენდპოინტებისთვის გამზადებული კონტროლერები, 
რომლებიც იღებენ inventory და location მოდელებთან დაკავშირებულ მოთხოვნებს და პასუხობენ შესაბამისად. 

### src/db
db ფოლდერი მოიცავს მონაცემთა ბაზასთან დაკავშირებულ დეტალებს, კერძოდ, დავალების მოთხოვნის მიხედვით განსაზღვრულ მოდელებს და მათ ასოციაციას, 
ასევე Sequelize–ის კონფიგურაციას. 

### src/middleware
middleware ქვედირექტორია ჯერჯერობით არ არის პასუხისმგებელი აპლიკაციასთან დაკავშირებულ ლოგიკაზე, მისი მიზანია მომავალში საჭიროების შემთხვევაში ენდპოინტებისთვის დახმარების გაწევა ისეთ საკითხებთან დაკავშირებით, როგორებიცაა
ავტორიზაცია და ავთენტიკაცია. 

### src/routes
routes იყენებს controllers-ში განსაზღვრულ მეთოდებს რათა შესაბამის URL–ზე მიღებულ მოთხოვნებს უპასუხოს, იგი მოიცავს ორ კომპონენტს, InventoryRoute.ts და LocationRoute.ts. 

### src/scripts
scripts ქვედირექტორია პასუხისმგებელია ბაზასთან დაკავშირებული მარტივი ოპერაციების შესრულებით, კერძოდ, იგი შედგება ორი სკრიპტისგან, inventorySeeder.ts და InventoryUnseeder.ts, აქედან პირველი გაშვების შემთხვევაში ბაზაში შექმნის 500000 სატესტო 
inventory მოდელის ნივთს, ხოლო მეორე წაშლის ამ 500000–სს. მათ გამოსაყენებლად საჭიროა `npx ts-node src/scripts/inventorySeeder.ts` და `npx ts-node src/scripts/inventoryUnseeder.ts` ბრძანებების გამოყენება.
